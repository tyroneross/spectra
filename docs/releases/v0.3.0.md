# v0.3.0 — First GitHub Packages release

First release of Spectra on GitHub Packages npm registry as `@tyroneross/spectra`.
Applies the [`publish-packages` skill](https://github.com/tyroneross/RossLabs-AI-Toolkit/blob/main/skills/publish-packages/SKILL.md).

## What ships in the tarball

Per `package.json` `files:` whitelist (tight — previously an unwhitelisted pack
would have shipped 565 files / 4 MB unpacked):

- `dist/` — compiled TypeScript (the runtime entry, MCP server, CLI)
- `bin/spectra` — CLI entry point
- `scripts/postinstall.sh` — post-install hook (see below)
- `.claude-plugin/` — Claude Code plugin manifest + MCP server config
- `.codex-plugin/` — Codex plugin manifest + MCP server config
- `commands/` — slash commands
- `skills/` — skill definitions
- `README.md`, `LICENSE`

Excluded from the tarball: TS source (`src/`), tests, `web-ui/` (the
dashboard, separate runtime), `macos/` Xcode project, `archive/`,
`artifacts/`, and the binary `Spectra.app` + `Spectra.dmg` at repo root.

## What changed (vs. previous tagged `v0.1.0`)

This is the first release wired for npm publishing; the prior tag was
purely a marketplace marker.

- **Renamed** `package.json` `name` from `spectra` → `@tyroneross/spectra` so
  it can publish to GitHub Packages under the owner's scope.
- **Added `files:` whitelist** so the tarball is ~tight, not ~4 MB.
- Added `publishConfig` pointing the publish target at
  `https://npm.pkg.github.com` with `access: public`.
- Synced `.codex-plugin/plugin.json` version `0.2.0 → 0.3.0` to match
  `.claude-plugin/plugin.json` and `package.json`.
- New `.github/workflows/publish-npm.yml` — release-triggered publish with
  `npm ci --ignore-scripts` + `npm run build` (produces `dist/`) before
  `npm publish`. Tag-vs-manifest drift guard + inter-manifest drift guard.
- New `.github/workflows/verify-install.yml` — chained smoke test that
  asserts dist entrypoints exist, `.claude-plugin` + `.codex-plugin`
  manifests landed, and the package actually imports under Node 22 ESM.

## Install for consumers

```bash
echo "@tyroneross:registry=https://npm.pkg.github.com" >> ~/.npmrc
echo "//npm.pkg.github.com/:_authToken=<PAT>" >> ~/.npmrc

# Skip the postinstall on macOS if you don't want the dist mirror:
npm install --ignore-scripts @tyroneross/spectra
```

## Postinstall behavior (macOS only)

`scripts/postinstall.sh` runs on `npm install` on macOS. It does two things:

1. **DMG prompt** — only if `CLAUDE_PLUGIN_INSTALL=1` is set in the env
   (Claude Code marketplace install path). Otherwise no-op.
2. **Dist mirror** — copies the freshly-installed `dist/` into
   `~/.spectra/dist/` so the launchd daemon can find it without
   resolving through plugin-cache paths.

On Linux/Windows the script exits early at the OS check. Consumers who
don't want the mirror can use `npm install --ignore-scripts`.

## No skill / runtime changes

This release is packaging only. Spectra behavior, MCP tools, slash commands,
and skill content are unchanged from the previous `0.3.0` Claude Code
marketplace state.
